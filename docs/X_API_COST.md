# X API Cost Structure

## Pricing Model (2026-03)

X APIは従量課金 (Pay-per-usage)。旧Tier制 (Free/Basic/Pro) は廃止済み。
- クレジットを事前購入 (最低$5)、リクエスト毎に消費
- 同一リソースは24時間内1回分の課金 (重複排除)
- 月間上限: 200万Post読み取り (Pay-per-usage)
- X APIクレジット購入でxAI APIクレジット最大20%還元

## 実測コスト (2026-03-29)

課金は取得件数に比例する。クエリ数ではなく返却ツイート数が主要コスト要因。

| テスト | クエリ数 | 取得件数 | コスト | 1件あたり |
|--------|---------|---------|--------|----------|
| 検索 (10件指定) | 1 | 10件 | $0.05 | $0.005 |
| キーワード検索 (100件指定) | 5 | 31件 | $0.06 | ~$0.002 |
| from: OR検索 (100件指定) | 3 | 299件 | $1.47 | ~$0.005 |

1ツイートあたり約 $0.002-0.005 が実コスト。

## 監視コスト月額見積もり (ハイブリッド方式)

| 方式 | 頻度 | 推定取得件数/月 | 月額 |
|------|------|---------------|------|
| キーワード検索 (日次) | 30日 x 5クエリ | ~4,500件 | $9-22 |
| from: OR検索 (月2回) | 2回 x 3バッチ | ~1,800件 | $4-9 |
| **合計** | | | **$13-31** |

## コスト最適化

- max_results を減らす (100 -> 20-30) でコスト削減可能
- 検索結果が少ないクエリは安い (返却件数比例)
- 失敗クエリ (400エラー) は返金される模様
- from: OR検索は1クエリで最大23アカウントをまとめられる (512文字制限)
